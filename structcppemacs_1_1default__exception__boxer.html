template<bool Box = CPPEMACS_ENABLE_EXCEPTION_BOXING>
struct cppemacs::default_exception_boxer< Box >

The default exception boxing type.

This implements the default behavior described by envw::run_catching(), envw::maybe_non_local_exit(), and envw::rethrow_non_local_exit(), which is documented on those methods.

This type defines the three methods which are used to control the exception behavior. Clients can derive from this type to customise the behavior.

See also
CPPEMACS_ENABLE_EXCEPTION_BOXING and CPPEMACS_DEFAULT_EXCEPTION_BOXER which can be used to override the default exception boxing behavior.

Raise a C++ exception based on the current non-local exit status.

This is called by envw::maybe_non_local_exit() when a envw::non_local_exit_check() is true. It is expected to raise a C++ exception, but may or may not clear the non-local exit status.

Clear and raise the current non-local exit, preserving the data.

This is called directly by envw::rethrow_non_local_exit() to throw the exception to C++. This should return normally if no non-local exit is pending.

Handle the current C++ exception (from a catch block), and re-raise it to Emacs.

This is called by envw::run_catching() to catch the exception. The environment must have a non-local exit pending when this returns, or the program will abort.
