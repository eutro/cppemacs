◆ CPPEMACS_SUPPRESS_WCOMPAT_MANGLING_BEGIN

#define CPPEMACS_SUPPRESS_WCOMPAT_MANGLING_BEGIN

On Clang, begin suppressing -Wc++17-compat-mangling.

Procedures which take emacs_function or emacs_finalizer as parameters should have them annotated with noexcept in order to match the C types, but before C++17 these are not (in the standard) part of the type system, which breaks name mangling.

This warning can safely be ignored, since all affected procedures are inline, and no attempt is made to preserve binary compatibility.

This must be paired with a matching CPPEMACS_SUPPRESS_WCOMPAT_MANGLING_END.

◆ EMACS_MAJOR_VERSION

#define EMACS_MAJOR_VERSION   29

The compile-time major version of Emacs.

This is defined in emacs-module.h since Emacs 27, and falls back to 25 if that is not found. The value shown in documentation is the highest that cppemacs supports.

If you might be on exactly Emacs 26 and desperately need envw::should_quit(), define the secret CPPEMACS_MAJOR_VERSION_FALLBACK as 25 or 26 before including this file, and please tell me! I'd like to know what you're doing.

See also
envw::is_compatible() for checking runtime version compatibility.

◆ emacs_function

using cppemacs::emacs_function = typedef value (*)(emacs_env *env, ptrdiff_t nargs, value *args, void *data) noexcept

A raw module function, for envw::make_function().

Warning
Type aliases like this one and emacs_finalizer can only be noexcept in C++17 and above (P0012R1), but function parameters can be noexcept even in C++11. Be careful using this as a type that is forwarded to envw::make_function(), as it will not typecheck on certain compilers. Write out the full function type instead, and surround it with CPPEMACS_SUPPRESS_WCOMPAT_MANGLING_BEGIN.
See also
make_module_function() and make_spreader_function() for convenience functions that produce emacs_functions from C++ closures.

◆ emacs_runtime

using cppemacs::emacs_runtime = typedef ::emacs_runtime

Struct passed to emacs_module_init().

This struct has two (public) fields:

  • ptrdiff_t size, the size, in bytes, of emacs_runtime in the running Emacs binary.
  • emacs_env *(*get_environment)(emacs_runtime *runtime), which returns an emacs_env that can be used until emacs_module_init() returns.
extern "C" int emacs_module_init(emacs_runtime *rt) noexcept {
if (rt->size < sizeof(*rt)) return 1; // check size (though it is unlikely to change)
envw env = rt->get_environment(rt);
// ...
}

◆ emacs_module_init()

int emacs_module_init ( cppemacs::emacs_runtime *  runtime)
noexcept

The main entrypoint of the dynamic module. See the \manlink{manual|Module-Initialization.html::index-emacs_005fmodule_005finit-1}.

Parameters
runtimeThe emacs_runtime, which can be used to obtain an emacs_env to access the module API.
Returns
The exit status. If this is non-zero, Emacs will @manlink{raise a signal|Signaling-Errors.html::index-signal} module-init-failed with it.
using namespace cppemacs;
extern "C" int emacs_module_init(emacs_runtime *rt) noexcept {
// check size (though it is unlikely to change)
if (rt->size < sizeof(*rt)) return 1;
envw env = rt->get_environment(rt);
// check the size of the environment
if (!env.is_compatible<27>()) return 2;
// ...
}
