Make a module_function with the given minimum and maximum arity.

Parameters
min_arityThe minimum number of arguments the function can be invoked with.
max_arityThe maximum number of arguments the function can be invoked with.
If this is emacs_variadic_function, the function can be called with any number of arguments >= min_arity.
docThe documentation string of the function. See the manual. In particular, a (fn arglist) line at the end is helpful to provide argument names, since they cannot be inferred from the source code.
fThe implementation of the function, which will be called with a new environment and the arguments. It must be invocable with
(emacs_env *env, ptrdiff_t nargs, value *args) for min_arity <= nargs <= max_arity, and return a value. It will be wrapped with envw::run_catching().
Returns
A module_function<F>, suitable for conversion to an Emacs function.
See also
make_spreader_function() which automatically unpacks the arguments.

Make a spreader module function with the given minimum and maximum arity.

Returns a module_function that can be called with at least MinArity arguments. Arguments are passed as cells, and the return type is converted .

Warning
The function must return a to-Emacs convertible type (not void). Despite my best efforts, this can generate incomprehensible errors, particularly if used without C++20 concepts.

With IsVariadic false, f will be invoked with emacs_env followed by exactly MaxArity arguments:
(emacs_env *env, A_1 arg_1, ..., A_MaxArity arg_MaxArity). Arguments that are provided by the caller will be given as cells, whereas absent arguments will be default-initialized. On the Emacs side, the function cannot be invoked with more than MaxArity arguments.

With IsVariadic true, the resulting function can be called with more than MaxArity arguments, and f is additionally invoked with a spreader_restargs argument containing the remaining arguments:
(emacs_env *env, A_1 arg_1, ..., A_MaxArity arg_MaxArity, spreader_restargs rest).

See also
cell_extracted, which automatically converts function arguments to the desired type.

Examples

static void spreader_function_examples(envw env) {
env->*make_spreader_function(
spreader_thunk(),
"Do something with zero arguments.",
[](envw env) {
// ...
return nullptr;
});
env->*make_spreader_function(
spreader_arity<1, 2>(),
"Do something with X and maybe Y.\n\n(fn X &optional Y)",
[](envw env, value x, value y) {
if (x) {
// invoked with two arguments
} else {
// invoked with one argument
}
return nullptr;
});
// variadic functions
env->*make_spreader_function(
spreader_variadic<1>(),
"Do something with X and REST\n\n(fn X &rest REST).",
[](envw env, value x, spreader_restargs rest) {
for (value val : rest) {
// ...
}
return nullptr;
});
env->*make_spreader_function(
spreader_variadic<1>(),
"Do something with X and REST\n\n(fn X &rest REST).",
[](envw env, value x, std::vector<value> rest) {
// ...
return nullptr;
});
#if defined(__cpp_lib_span)
env->*make_spreader_function(
spreader_variadic<1>(),
"Do something with X and REST\n\n(fn X &rest REST).",
[](envw env, value x, std::span<value> rest) {
// ...
return nullptr;
});
#endif
}

In-place construct a user_ptr on the heap.

Warning
If this value isn't passed to Emacs via envw::inject(), then it will leak memory. The suggested idiom is:
env.maybe_non_local_exit();
env->*make_user_ptr<T>(a, b, c);
which will ensure that no allocation is made if there is already a non-local exit pending.
